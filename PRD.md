# PRD — 반도체 시료 생산주문관리 시스템

## 배경 / 문제

가상의 반도체 회사 "S-Semi"는 다양한 반도체 시료(Sample)를 생산해 연구소, 팹리스 업체, 대학 연구실 등에 납품한다. 기존에는 엑셀/메모장으로 주문을 관리해 실수가 잦고 재고·공정 현황을 한눈에 파악하기 어려웠다. 콘솔 기반의 통합 관리 시스템으로 이를 해결한다. (상세 배경: `CLAUDE.md` 참고)

## 목표

기능 명세(`CLAUDE.md`)를 모두 충족하는 콘솔 애플리케이션을, TDD Red-Green-Review 사이클과 Agentic Engineering 방법론에 따라 고품질로 구현한다.

## 사용자

| 역할 | 설명 |
|---|---|
| 고객 | 필요한 시료를 이메일 등으로 요청하는 시료 요청자. 시스템을 직접 사용하지 않고, 주문 담당자를 통해 간접적으로 상호작용한다. |
| 주문 담당자 | 고객의 시료 요청을 받아 요청에 맞게 주문서를 작성(주문 접수)하고, 시스템에 시료를 등록·조회하며, 출고 처리와 모니터링을 수행한다. |
| 생산 담당자 | 개발 시료 등록, 주문 수신 후 승인/거절 처리, 생산라인 운영 확인을 담당한다. |

역할별 흐름: 고객이 시료를 요청 → 주문 담당자가 주문서 작성·전달 → 생산 담당자가 승인/거절 → 주문 담당자에게 결과 전달.

(콘솔 단일 애플리케이션이므로 실제 로그인/권한 분리는 없음 — 메뉴로 역할별 기능만 구분. 고객은 시스템 사용자가 아니라 요청의 출발점으로만 존재한다.)

## 핵심 시나리오 (End-to-End)

1. 주문 담당자가 시료를 시스템에 등록한다 (ID, 이름, 평균 생산시간, 수율).
2. 고객 요청에 따라 주문 담당자가 시료 주문을 접수한다 → 주문 상태 `RESERVED`.
3. 생산 담당자가 대기 중인 주문을 승인한다.
   - 재고가 충분하면 즉시 `CONFIRMED`.
   - 재고가 부족하면 부족분만큼 생산라인에 등록되고 `PRODUCING`.
4. 생산라인은 FIFO로 대기 주문을 순서대로 생산하고, 완료되면 주문을 `CONFIRMED`로 전환하며 재고를 갱신한다.
5. 주문 담당자가 `CONFIRMED` 주문을 출고 처리하면 `RELEASE`로 전환된다.
6. 언제든 모니터링 메뉴에서 상태별 주문 현황과 시료별 재고 현황(여유/부족/고갈)을 확인할 수 있다.

## 도메인 규칙

### 주문 상태

`RESERVED → (승인) → CONFIRMED | PRODUCING → (생산 완료) → CONFIRMED → (출고) → RELEASE`
`RESERVED → (거절) → REJECTED` (모니터링 제외)

### 수율(Yield) 규칙과 재고 반영 — 예시

수율은 `정상 생산 시료 수 / 총 생산 시료 수`로 정의되며, 부족분을 생산할 때 수율만큼 여유 있게 생산량을 계산하는 데 사용된다. **본 시스템에서는 실제 반도체 공정의 불량 개념이 아니라, "생산량 대비 실제 주문 충족량의 비율이며 남는 초과분은 폐기되지 않고 재고로 누적되는 규칙"으로 단순화한다.**

예시: 수율 80%(0.8)인 시료에 재고 0 상태로 16개 주문이 들어온 경우

| 단계 | 값 |
|---|---|
| 주문 수량 | 16 |
| 재고 | 0 |
| 부족분 | 16 (= 주문 수량 − 재고) |
| 실생산량 | ceil(16 / 0.8) = **20** |
| 총 생산 시간 | 평균 생산 시간 × 20 |
| 생산 완료 시 재고 반영 | 재고 0 → **20** (생산량 전부가 재고로 들어감, 부분 완료 없이 20개가 한 번에 다 차야 완료) |
| 주문 소비 | 주문 16개 충족 → 주문 상태 `PRODUCING` → `CONFIRMED` |
| 남는 재고 | 20 − 16 = **4** (잉여 재고로 다음 주문을 위해 남음) |

즉, 부족분 계산과 실생산량 계산은 항상 "이 주문 하나를 확실히 채우기 위한 최소 생산량"을 수율을 반영해 넉넉하게 산출하며, 그 결과 생기는 나머지는 버려지지 않고 그대로 재고에 누적된다.

### 재고 상태 표기 (모니터링)

- 여유: 주문 대비 재고 충분
- 부족: 주문 대비 재고 수량 부족
- 고갈: 재고 수량 0

## 완료 기준 (Definition of Done)

- [ ] `CLAUDE.md`의 6개 기능 명세(시료 관리, 시료 주문, 주문 승인/거절, 모니터링, 생산라인, 출고 처리)가 모두 콘솔에서 동작한다.
- [ ] 도메인 로직(Model)에 대한 pytest 테스트가 모두 통과하며, 각 테스트는 RED → GREEN → REVIEW 사이클을 거쳐 작성되었다.
- [ ] 수율/부족분/실생산량 계산 규칙이 위 예시와 일치하게 구현·테스트되어 있다.
- [ ] Clean Code 원칙(YAGNI, 역할 분리) 준수, 커밋 이력에 RED/GREEN이 분리되어 있다.
