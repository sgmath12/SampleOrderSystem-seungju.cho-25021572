# 전체 플랜 — 반도체 시료 생산주문관리 시스템

`CLAUDE.md`의 기능 명세를 Agentic Engineering 방법론(문서 관리 / Harness / TDD Red-Green-Review / Clean Code / Commit 이력)에 따라 구현하기 위한 전체 로드맵.

## 개발 원칙

- 아래 마일스톤을 사이클 단위로 쪼개고, 사이클마다 `plan/plan_N.md`를 먼저 작성해 사람 파트너 검토를 받은 뒤 RED → GREEN → REVIEW 순서로 진행한다.
- 한 사이클 = 하나의 좁은 목표. Plan에 없는 범위는 다음 사이클로 미룬다.
- 도메인 로직(Model)은 순수 함수/클래스로 작성해 pytest로 검증하고, 콘솔 입출력(View)은 도메인 로직에서 분리해 테스트 가능한 구조를 유지한다 (ConsoleMVC POC 참고).

## 마일스톤

### M0. 프로젝트 뼈대 & Harness

- 패키지 구조 설계 (model / view / controller, 또는 domain / cli)
- pytest 실행 환경 구성 (`pytest.ini` 또는 `pyproject.toml`, 테스트 폴더 `tests/`)
- `PRD.md` 작성

### M1. 시료 관리 (Sample)

- 시료 등록 (ID, 이름, 평균 생산시간, 수율)
- 시료 조회 (재고 포함), 시료 검색 (이름 등)
- 등록된 시료만 주문 가능하다는 제약 검증

### M2. 시료 주문 (Order 생성)

- 주문 접수 → RESERVED 상태로 생성 (시료 ID, 고객명, 수량 입력)
- 존재하지 않는 시료로 주문 시 에러 처리

### M3. 주문 승인/거절

- RESERVED 목록 조회
- 승인 시: 재고 충분 → CONFIRMED / 재고 부족 → PRODUCING + 생산 큐 등록
- 거절 시: REJECTED

### M4. 생산라인 (FIFO)

- 생산 큐 (FIFO) 관리, 부족분 계산 (실생산량 = ceil(부족분 / 수율), 총 생산시간 = 평균 생산시간 × 실생산량)
- 생산 완료 처리 → PRODUCING → CONFIRMED, 재고 반영
- 현재 생산 중 / 대기 목록 조회

### M5. 출고 처리

- CONFIRMED 주문에 대해 출고 실행 → RELEASE 전환, 재고 차감

### M6. 모니터링

- 상태별 주문 수 집계 (REJECTED 제외)
- 시료별 재고 현황 (여유/부족/고갈)

### M7. 콘솔 UI 통합

- 메인 메뉴 및 각 화면 콘솔 입출력 연결 (View/Controller)
- 전체 시나리오 수동 검증 (요청 → 승인 → 생산 → 출고)

### M8. 마무리

- 전체 테스트 스위트 정리, Clean Code 리뷰
- README 작성, 최종 커밋 정리

## Plan 파일 관리

- `plan/plan_1.md`부터 순번대로 생성. 각 파일은 하나의 RED-GREEN-REVIEW 사이클 목표를 기술한다.
- 사이클 완료 시 본 문서의 해당 마일스톤에 진행 상태를 표시한다.
